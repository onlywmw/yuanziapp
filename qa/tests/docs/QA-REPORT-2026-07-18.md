## 四、修改建议（按优先级）

### 立即处理（阻塞 M2）

1. **BUG-007 文件读取沙箱**：atom-file-read 增加 `allowed_roots` 配置，对 `os.path.realpath(path)` 做前缀校验，拒绝 `..` 穿越；默认仅允许白名单目录。
2. **BUG-008 SSRF 防护**：atom-http-get 增加协议白名单（仅 http/https）+ 解析后 IP 检查（拒绝 127.0.0.0/8、10/8、172.16/12、192.168/16、169.254/16）；文档注明可信网络例外。

### 本周内

3. **BUG-009 绑定与认证**：所有原子与模板默认绑定 `127.0.0.1`；模板 server.py 增加可选 `YUANZI_TOKEN` 校验（与 Android 端 `Yuanzi-Token` header 对应）；需要对外暴露时在文档中给出加固步骤。
4. **BUG-005 端口冲突**：atoms/*/server.py 改为从 meta.yaml `runtime.port` 或 `PORT` 环境变量读取；5 个示例分配 9000-9100 段内不同端口（与 yuanzi-config.yaml 的 default_port_range 对齐）。
5. **BUG-006 签名去重**：`compute_signature` 从载荷中移除 `atom_id`（身份唯一性已由 atom_id UNIQUE 约束保证；签名应只描述能力），并为 TC-REG-002 场景补一条单元测试。
6. **BUG-001 init 校验**：`run_init` 在调用 cookiecutter 前复用 `meta.validate_atom_id`；非法 id 直接 `typer.Exit(1)` 并给出示例格式。

### 下个迭代

7. **BUG-002**：捕获 cookiecutter 异常转友好提示；增加 `--overwrite` 选项。
8. **BUG-004**：test_validate.py 改用 `Path(__file__).resolve().parents[2]` 锚定（与 test_init.py 一致）。
9. **BUG-010**：模板 `_read_body` 增加 `MAX_BODY_BYTES`（如 5MB），超限返回 413。
10. **BUG-012**：异常对外返回通用错误码，详情只写服务端日志；可增加 `--debug` 开关。
11. **BUG-003**：id 校验改为显式 ASCII 小写正则 `^[a-z0-9][a-z0-9_-]*(\.[a-z0-9][a-z0-9_-]*)+$`。
12. **BUG-011**：adb 路径拆到未跟踪的 `yuanzi-config.local.yaml`；Android 端改 EncryptedSharedPreferences。
13. **BUG-013**：引入 uv/pip-tools 生成 lock；CI 接入 pip-audit。

### 工程实践建议（非缺陷）

- **CI 缺失**：仓库只有本地 pre-commit，建议加 GitHub Actions（lint + CLI 测试 + 示例原子测试 + pip-audit）。
- **代码覆盖率**：建议接入 pytest-cov，目标 ≥80%，当前未量化。
- **一致性**：atoms `/meta` 的 id 风格（`atom.file.read`）与 CLI/模板的反向域名惯例（`com.example.sum`）不统一，建议生态规范文档化。
- **atoms 无访问日志**：建议复用模板 server.py 的 `log_message` 风格补访问日志。
- **math-sum 溢出**：`1e308+1e308` 返回 `Infinity` 且 status=success，建议校验输入范围或标记溢出。
- **测试隔离**：示例原子 test_health.py 固定端口 18777，建议改动态端口防冲突。
- **v2 功能未实现**：meta.py 的 `kernel_type` 支持 markdown_rules/prompt_txt，但模板 server.py 对非 python_script 直接 raise NotImplementedError——validate 通过的 markdown_rules 原子无法运行，建议要么实现要么在 validate 阶段提示。

## 五、结论

核心功能质量良好：**CLI 三个命令主流程全部可用，已有 12 条测试全绿，注册中心的提交/审核/状态机/审计逻辑正确且防注入**。主要风险集中在**安全基线**（示例原子的文件读取与 HTTP 抓取无防护、服务对外暴露无认证）和**若干工程细节**（init 校验缺失、端口冲突、签名去重失效）。
