# 原子可见性模型

> **核心**: 不是所有原子都需要在首页展示。原子自己决定自己的可见度——soul 饱满的上浮, 空心的下沉。
> **原则**: 不打标签, 不人工分类。让原子的完整度自然决定它出现在图谱的哪一层。

---

## 一、自然浮现

```
soul 饱满 (叙事+风格+受众)  ──→  浮到上层, 节点更大
soul 半满 (有部分字段)      ──→  中层, 正常大小
soul 空白 (纯功能描述)      ──→  底层, 节点缩小
system.* (基础原子)         ──→  不可见, 不可操作

像水里的气泡——饱满的上浮, 空心的下沉。
```

---

## 二、浮现算法

```python
def visibility_score(atom):
    score = 0.0
    soul = atom.get("classification", {}).get("soul") or {}

    # soul 完整度加分
    if soul.get("narrative"):     score += 3.0
    if soul.get("style"):         score += 1.5
    if soul.get("audience"):      score += 1.0
    if soul.get("mood"):          score += 0.5
    if soul.get("quality"):       score += 0.5

    # 社区加分
    ratings = atom.get("ratings", {})
    if ratings.get("count", 0) >= 10:  score += 1.0
    if ratings.get("average", 0) >= 4.0: score += 0.5

    # 系统原子强制归零
    if atom["atom_id"].startswith("system."):
        score = 0.0

    return score   # 0.0 (不可见) ~ 7.5 (最亮)
```

### 分数 → 渲染

```
score ≥ 4.0:    展示层 — 正常节点, 首页可见
score 1.0~3.9:  折叠层 — 缩小节点, "显示全部"后可见
score < 1.0:    隐藏层 — 不绘节点, 搜索或作为依赖时出现
score = 0:      不可见 — system.*, 永不绘制
```

### 一个原子的成长过程

```
刚注册 (soul 为空):
  score = 0 → 隐藏层, 搜索能找到, 首页看不到

作者填写 soul:
  score = 1.5 → 折叠层, "显示全部"可见

社区开始使用, 评分积累:
  score = 3.0 → 折叠层, 节点变大

叙事丰满, 高分好评:
  score = 6.0 → 展示层, 首页可见, 正常大小
```

## 三、图谱渲染规则

```
默认视图:
  只渲染 score ≥ 4.0 的原子
  → 首页干净, 只有真正饱满的原子

"显示全部" 模式:
  渲染 score ≥ 1.0 的原子
  system.* 仍然不渲染

搜索时:
  无视 score, 所有匹配结果都显示
  搜索结果关闭后恢复默认视图

局部图谱:
  双击原子 → 显示其邻居
  即使邻居 score 低 → 也显示 (因为是依赖关系)
  让基础设施在"被需要的时候"才出现
```

## 四、首页效果

```
默认 (score ≥ 4.0):
  ┌──────────────────────────────────────┐
  │                                      │
  │    [原子 A]        [原子 B]          │  ← 5-8 个饱满的原子
  │         [原子 C]                     │
  │                                      │
  │  干净, 不拥挤, 每个都值得看          │
  │                                      │
  │  [显示全部 61 个原子]  ← 底部开关    │
  └──────────────────────────────────────┘

显示全部 (score ≥ 1.0):
  ┌──────────────────────────────────────┐
  │  更多节点出现                         │
  │  低 score 的节点更小更淡              │
  │  高 score 的节点保持醒目              │
  │                                      │
  │  [只显示精选]  ← 底部开关            │
  └──────────────────────────────────────┘
```

## 五、为什么不用标签

```
标签方案 (terminal/infrastructure):
  管理员手动分类
  作者自己判断
  分类不准确就没意义
  多了一个要维护的字段

浮现方案 (自动计算):
  soul 写得好 → 自然上浮
  社区喜欢 → 自然上浮
  什么都不写 → 自然下沉
  不需要任何新字段
```

---

## 六、零改动

```
不新增 schema 字段
不新增数据库列
不新增 API 参数
不需要管理员介入

只加一个函数: visibility_score(atom) → float
GraphEngine 读这个分数决定渲染层级
```
