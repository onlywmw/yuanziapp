# 注册原子规则

> **性质**: 指导性规范 — 注册原子必须遵守的最低标准
> **原则**: 规则约束结构，内容保持开放

---

## 必填字段

| 字段 | 说明 | 为什么必须 |
|------|------|-----------|
| `atom_id` | 全局唯一标识，反向域名风格 | 身份 |
| **`author`** | **作者姓名或组织名** | **连接世界的人** |
| `name` | 人类可读名称 | 展示 |
| `description` | 一句话简介 | 发现 |
| `purpose.summary` | 这个原子解决什么问题 | 搜索匹配 |
| `purpose.functions` | 提供的具体功能列表 | 能力声明 |

## 关于作者

**作者是原子生态中最重要的字段。**

```
为什么:
  · 61 个原子背后是真实的开发者
  · 作者让能力有了归属
  · 人通过创造的工具连接彼此
  · 未来原子市场中的评分、信任、协作都基于作者

规则:
  · author 不可为空
  · author 不可伪造
  · author 一经注册不可更改（版本升级保留原作者）
  · 尊重署名，就是尊重创造
```

**作者字段在 schema 中的位置:**

```json
{
  "ownership": {
    "author": "张三",              ← 必填
    "maintainer": "张三",          ← 可选，默认同 author
    "organization": "Yuanzi",      ← 可选
    "license": "MIT",
    "source_url": "https://..."
  }
}
```

## 推荐字段

| 字段 | 说明 |
|------|------|
| `version` | semver，默认 1.0.0 |
| `classification.tags` | 有助于搜索发现 |
| `purpose.examples` | 使用示例让人更快理解 |
| `architecture.dependencies` | 声明依赖的其他原子（不强制引用基础原子名） |

---

## 不强制

- **不强制声明对基础原子的依赖**：注册原子可以自由调用任何能力，依赖关系由架构声明，不由系统强制
- **不限制实现语言**：Python / Node / Go 均可，只要实现三个标准端点
- **不限制部署方式**：Docker / 进程 / Lambda 均可，声明在 `architecture.hosting`

---

## 禁止

- **禁止伪造作者**
- **禁止包含恶意代码**（probe 机制持续验证可达性）
- **禁止重复注册**（签名机制自动去重）

---

## 注册流程

```
开发者创建原子
  → 实现 /health /meta /run 三个端点
  → 填写 atom_id / author / name / description / purpose
  → 提交注册
  → 系统自动计算签名 (SHA-256)
  → 自动分类
  → 审核（未来：社区投票）
  → 注册成功 → 知识图谱可见
```
