# 体验场景优化

> **方法**: 站在不同人的视角, 走完他们使用 Yuanzi 的一天
> **目标**: 找出每个场景下还缺什么

---

## 场景一: 第一次打开的人

```
我下载了 Yuanzi, 第一次打开。

看到的: 深色背景, CENTER 节点像一颗恒星在呼吸。周围空荡荡的。

问题: 然后呢？
```

**缺**:

```
1. 引导式初始化
   "你是什么类型的人？"
   [开发者] [创作者] [普通用户] [极客]
   → 系统推荐第一批原子

2. 预置工作流模板
   "试试这些:
    · 每日天气 + 音乐推荐
    · 文件备份助手
    · 咖啡厅模式"
   一键安装, 图谱立刻出现第一批节点

3. 空白状态的美
   不是"空", 是"等待探索"
   CENTER 节点呼吸, 周围偶尔飘过微小的光粒子
   像宇宙大爆炸前的那一刻
```

---

## 场景二: 工作流分享

```
我在咖啡厅发现了一个很棒的工作流:
  位置 → 天气 → 设备 → context-fusion → 推荐音乐

我想分享给朋友。
```

**缺**:

```
4. 一键分享工作流
   在工作流画布上点 [分享]
   → 生成链接: yuanzi.app/wf/abc123
   → 朋友打开 → 一键安装到自己的图谱
   → 自动适配朋友的设备 (如果他没有某原子, 提示安装)

5. 工作流市场
   不只是原子的市场, 还有工作流模板的市场
   "咖啡厅模式" · 128 人使用 · ⭐4.8
   "晨间仪式"   · 89 人使用  · ⭐4.6
   "极客桌面"   · 56 人使用  · ⭐4.9
```

---

## 场景三: 信任与安全

```
市场里有一个原子: "超级音乐推荐"
描述很炫, 评分很高, 但我不知道它背后做了什么。
```

**缺**:

```
6. 权限透明度
   安装前显示:
     "这个原子将访问:
      ✅ 你的位置 (用于推荐附近活动)
      ✅ 你的天气 (用于匹配氛围)
      ❌ 不访问: 通讯录、相册、麦克风"

7. 安全信号
   原子详情面板:
     ✅ 签名验证通过
     ✅ 作者实名认证
     ✅ 开源 (可查看代码)
     ⚠️ 使用第三方 API
     ❌ 未认证作者

8. 原子行为日志
   安装后, 可查看原子做了什么:
     "今天: 读取位置 3 次, 查询天气 1 次, 从未访问相册"
```

## 场景四: 时间与地点

```
有些原子只在特定时候有用。

早上 7 点 → 想看到"晨间仪式"相关的原子
到了咖啡厅 → 想看到"咖啡厅模式"
晚上 11 点 → 需要"睡眠模式"
```

**缺**:

```
9. 时间感知
   原子可以绑定时间段:
     "只在 6:00-9:00 激活"
     "只在周末可见"
     "只在晚上降低亮度"

   图谱随时间变化 — 早上看到的和晚上看到的不一样。

10. 地点感知
    原子可以绑定地理围栏:
      "进入咖啡厅 500m → 激活咖啡厅模式"
      "离开家 → 自动备份"
      "进入公司 → 切换到工作模式"

    图谱随地点变化 — 在咖啡厅看到的和在家看到的不一样。
```

## 场景六: 原子之间的对话

```
不是人操作原子 — 是原子自己对话。

天气感知检测到下雨
  → 通知 context-fusion
    → context-fusion 决定"这是忧郁的雨天"
      → 通知 rule-engine
        → rule-engine 决定"播放雨天歌单"
          → 通知 music-player 和 notification

这个链条, 人能看到吗？
```

**缺**:

```
12. 原子活动视图
    在图谱上, 当原子之间在通信时:
      连线发光 (正在传输)
      节点短暂闪烁 (被触发)
      右下角出现活动指示器 "3 个原子正在协作"

    人可以看到 — 不是看日志, 是看图谱上的光在流动。
    像看星系的脉搏。
```

## 场景七: 人与人的连接

```
我喜欢张三做的原子。
他最近在做什么？
还有谁喜欢张三的原子？
他们也喜欢什么？
```

**缺**:

```
13. 关注作者
    在原子详情面板点 [关注张三]
    → 张三发布新原子 → 通知我
    → 张三的原子出现在图谱上时, 有特殊标记
    → "你关注的作者" 分组在市场页面

14. 品味相似的人
    "喜欢 mcp.postgres 的人也喜欢..."
    "和你品味相似的人: 李四, 王五"
    "他们也安装了这些原子: ..."

    不是算法推荐 — 是"通过原子, 找到品味相似的人"。
```

---

## 场景八: 离线与容错

```
我在飞机上, 没有网络。
```

**缺**:

```
15. 离线模式
    没有网络时:
      感知原子继续工作 (位置/设备/时间不需要网络)
      本地工作流继续运行
      联网原子标记为"等待网络恢复"
      网络恢复后 → 自动补执行

    CENTER 节点在网络断开时变为灰色
    恢复后渐变回白色
```

## 总结: 缺什么

```
P0 - 必须有:
  1. 新用户引导 + 预置工作流
  4. 工作流一键分享
  6. 权限透明度
  9. 时间感知
  11. 个人时间线
  12. 原子活动视图

P1 - 应该有:
  5. 工作流市场
  7. 安全信号
  10. 地点感知
  13. 关注作者
  15. 离线模式

P2 - 锦上添花:
  8. 行为日志
  14. 品味相似的人
```
