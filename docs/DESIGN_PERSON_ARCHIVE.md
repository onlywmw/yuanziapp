# 人的档案

> **定位**: 系统认识一个人的全部依据。本地存储, 不上传。
> **原则**: 档案不是填出来的。是和你一起生活, 慢慢长出来的。

---

## 一、档案长什么样

```
不是问卷。不是标签。不是"你喜欢什么音乐？"

档案是一个文件。在你的手机里。
30 天后, 它是你的形状。
365 天后, 它是你的历史。

{
  "rhythm":   你每天的时间形状
  "response": 什么让你停下, 什么让你划过
  "silence":  你没说的那些
  "people":   你通过原子连接的人
  "growth":   你变了多少
}
```

## 二、节奏 — 你的时间形状

```
不是"工作日"和"周末"。

是你自己的周期:

  {
    "mornings": "不要催。给一点安静的声音。慢慢开始。",
    "afternoons_cafe": "可以深一点。长一点。我有时间。",
    "evenings": "不要推送。我在收尾。轻一点。",
    "rain": "熟悉的。有故事的。不需要新的。",
    "sunny_weekend": "随便。今天试新的。"
  }

不是你告诉系统的。是系统看出来的。
```

## 三、回应 — 什么让你停下来

```
不是"喜欢爵士"。
是你对每个建议的微反应:

  {
    "accepted": ["慢爵士", "窗外雨声背景", "不被干扰的通知"],
    "adjusted": [{"推荐": "快歌", "实际": "调低音量"}],
    "ignored": ["今天推了一个新作者", "推了一个社交建议"],
    "saved": ["一个 45 分钟的爵士列表。你收藏了。没马上听。"]
  }

不是数据。是你的选择。
你收藏但没听的, 比你不喜欢的更有信息。
你在等一个对的时刻。
```

## 四、沉默 — 你没说的

```
  {
    "不做": ["不接视频", "不在这个咖啡厅打电话", "晚上不碰工作"],
    "不说不": ["妹妹的电话。你从不拒绝。"], 
    "不打开": ["周末不看新闻", "下午不碰社交"]
  }

系统学到的:
  有些事情你不做, 不等于不喜欢。
  只是不在这里做。不在这个时候做。
```

## 五、人 — 你通过原子连接的人

```
  {
    "作者": [
      {"name": "张三", "reason": "他的原子总是极简的。从来不花哨。"},
      {"name": "李四", "reason": "每次更新都写在叙事里。你每次都读。"}
    ],
    "你关注的人": ["妹妹", "王五"],
    "和你相似的人": []  ← 本地, 不上传, 慢慢发现
  }
```

## 六、成长 — 你变了多少

```
  {
    "3 个月前": "你只听爵士。不接受推荐。降噪一直开着。",
    "现在": "你偶尔试新东西。降噪有时候关。你开始收藏了。"
  }

不是系统"优化"了你。是你自己在变。
系统只是注意到了。
```

## 七、档案的生命

```
第 1 天: 空的。系统什么都不了解。靠默认规则。
第 7 天: 有了节奏的轮廓。知道什么时候不该打扰你。
第 30 天: 知道你的形状。你的沉默。你的收藏。
第 90 天: 知道你在变。在往哪个方向变。
第 365 天: 档案就是你。不是"关于你"的数据。是你自己。
```

## 八、隐私

```
档案只在这台设备上。
不上传。不分析。不共享。
换手机了 → 导出 → 导入 → 还是你的档案。

如果删了 App → 档案也删除。
系统重新认识你。从零开始。
和第一次一样。
```
