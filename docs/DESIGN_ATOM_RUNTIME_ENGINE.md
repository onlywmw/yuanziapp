# 原子运行时引擎

> **定位**: 整个生态的底座。负责原子的发现、安装、调度、执行、销毁。
> **为什么缺它**: 没有运行时引擎, 原子只是 JSON 定义, 共振只是数学计算。

---

## 一、它做什么

```
原子发现:   本地基础原子 + 注册中心已安装 + 市场可安装
原子安装:   拉取原子定义 → 注册到本地 → 可被调用
原子调度:   决定什么时候调用哪个原子 (按需/持续/事件驱动)
原子执行:   调用 handler → 收集输出 → 传递给下游
原子销毁:   卸载 → 清理临时文件 → 释放资源
```

## 二、架构位置

```
┌─────────────────────────────────────┐
│  星云引擎 (共振/集群/输出)           │  ← 怎么组合
├─────────────────────────────────────┤
│  工作流引擎 (拓扑/执行/容错)         │  ← 怎么跑
├─────────────────────────────────────┤
│  运行时引擎 (发现/安装/调度/销毁)     │  ← 怎么活
├─────────────────────────────────────┤
│  原子定义层 (分类/基座/目录)         │  ← 是什么
└─────────────────────────────────────┘
```

## 三、原子生命周期

```
发现 → 安装 → 注册 → 运行 → 暂停 → 恢复 → 卸载

发现:   用户浏览市场 / 工作流自动匹配
安装:   拉取定义 + 依赖 → 本地注册
注册:   写入本地 atom_registry
运行:   按调度模式执行 (按需/持续/事件)
暂停:   用户禁用 / 系统省电
恢复:   重新启用
卸载:   清理 + 移除注册
```

## 四、调度模式

```
按需 (on-demand):   被工作流调用 → 执行 → 返回结果 → 休眠
持续 (continuous):  后台运行 → 定期采集 → 发出场 → 参与共振
事件 (event):       等待系统广播 → 收到事件 → 执行
```

## 五、依赖解析

```
安装原子时, 运行时引擎:
  1. 检查 architecture.dependencies
  2. 本地已安装 → 直接引用
  3. 未安装 → 自动从市场拉取
  4. 版本冲突 → 提示用户选择
  5. 平台不兼容 → 提示"此原子需要 Android 11+"
```

## 六、资源管理

```
临时文件 GC:
  工作流产生的临时文件 → 标记 TTL (默认 24h)
  到期自动清理
  工作流异常退出 → TTL 仍然生效 (不受影响)

内存:
  按需原子: 执行完释放
  持续原子: 有内存上限, 超过 → 暂停低优先级原子
```

## 七、实现

```
Android 端:
  McpService (已有) → 前台服务 + 通知栏
  负责持续原子的调度
  WorkManager → 定期采集
  BroadcastReceiver → 事件驱动

Chaquopy 内嵌:
  Python 端负责按需原子的执行
  handler 调用链全部在本地
```
