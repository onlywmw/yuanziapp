# 人原子 (Person Atom)

> **定位**: 人是系统中最重要、最复杂的原子。和其他原子一样参与共振, 但场更丰富。
> **原则**: 不分析人。不改造人。陪人走过去。

---

## 一、人 = 原子

```
person-atom 和 weather-atom 没有区别。

  都是发出场 → 和其他原子共振 → 形成集群。

  唯一的区别: person-atom 的场最丰富。
  因为它携带着一个人的全部——节奏、偏好、沉默、向往、痕迹。
```

## 二、两层场

人有两个场。同时发出。同时参与共振。

```
shown (展示的):
  人想成为的样子
  来源: 人自己说的, 自己选的, 展示给外界的
  例子: "我只听爵士"、"我是早起的人"、"我工作很专注"

actual (实际的):
  生活留下的真实痕迹
  来源: 行为、选择、时间留下的证据
  例子: 凌晨 2 点还在放嘻哈、11 点才起、下午 3 点开始刷手机

两层都是真的。
shown: 向往的方向。
actual: 此刻的位置。
```

## 三、共振权重

```
shown 和 actual 都参与共振, 但权重不同:

  差距大时:
    shown 权重 = 0.3  (系统不完全信展示的)
    actual 权重 = 0.7 (更相信痕迹)
    系统保持沉默, 只是观察, 偶尔在两点之间放一座桥

  差距缩小时:
    shown 权重 = 0.7 (展示的越来越可信)
    actual 权重 = 0.3
    系统轻轻靠近: "我注意到了"

  一致时:
    权重一样。不分 shown 和 actual。
    人不再需要展示层——他已经是他想成为的样子。
```

## 五、档案的生命

```
第 1 天:   空的。系统只靠默认规则。
第 7 天:   有了节奏的轮廓。知道什么时候不该打扰。
第 30 天:  知道你的形状。你的沉默。你的 gap。
第 90 天:  知道你在变。gap 在缩小还是扩大。
第 365 天: 档案就是你。shown 和 actual 越来越近。
```

## 六、工程接口

人原子和其他原子一样, 实现标准端点:

```
GET /health  → {"status": "ok"}
GET /meta    → {"atom_id": "person", "type": "sensor", ...}

输入 (持续自动采集, 不需要人调用):
  时间流:   时间戳 + 时段 + 星期 + 季节
  事件流:   应用使用、设备状态、蓝牙连接、位置变化
  交互流:   接受/忽略/跳过推荐

输出 (每 30s 更新一次, 供共振使用):
  {
    "shown":  {"早起": 0.9, "专注": 0.8},        ← 人展示的
    "actual": {"早起": 0.2, "下午刷手机": 0.7},   ← 实际痕迹
    "rhythm": {"pace": "slow", "push": false},    ← 当前节奏
    "gap":    {"早起": 0.7}                       ← 展示-实际差距
  }

隐私: 用户可关闭任意数据源的采集 (位置/蓝牙/应用使用)。
      关闭的数据源在档案中标记为 "user_disabled"。
```

## 七、隐私

```
档案只在这台设备上。
不上传。不分析。不共享。
换手机 → 导出 → 导入。
删 App → 档案也删除。

系统永远不做:
  · 比较 (你和别人, 昨天的你和今天的你)
  · 评判 (你应该...)
  · 说教 (早起对身体好)
  · 替你决定 (帮你关掉嘻哈)

系统只做一件事:
  接过你的展示, 记下你的实际, 在两点之间放一座桥。
```
