# 个性化 — 不是给你设定的, 是学你的

> **问题**: 工作流是死的。下雨→《Stan》只对我有用。
> **方案**: 决策原子不硬编码结果。每次执行后记录: 什么场景, 人做了什么选择。下次同样场景, 自动推荐人的偏好。

---

## 一、学, 不是设

```
现在: 作者预设规则
  IF rain + cafe + headphones THEN play "Stan"

好的: 系统学习偏好
  第 1 次: rain + cafe → 系统随机推荐 → 人选了《Stan》
  第 2 次: rain + cafe → 系统推荐《Stan》(上次选的) → 人确认
  第 3 次: rain + cafe → 人换成了爵士
  第 4 次: rain + cafe → 系统推荐爵士(最近选的)

  系统不是猜, 是记住。
  越用越懂你。
```

## 二、个人画像

```
不是全局推荐 "大家喜欢什么"。
只记录 "这个人喜欢什么"。

存储在本地, 不上传:

{
  "user": "张三",
  "contexts": [
    {
      "context": "rainy_cafe_headphones",
      "actions": [
        {"atom": "music", "choice": "Stan", "count": 7, "last": "2026-07-15"},
        {"atom": "music", "choice": "Lose Yourself", "count": 2, "last": "2026-07-10"}
      ]
    },
    {
      "context": "sunny_morning_home",
      "actions": [
        {"atom": "music", "choice": "爵士", "count": 12, "last": "2026-07-19"}
      ]
    }
  ]
}
```

## 三、决策流程

```
fusion 输出: "rainy_cafe_headphones"
    ↓
rule-engine:
  1. 查个人画像: 这个场景下, 历史选择是什么?
  2. 有历史 → 推荐历史选择 (置信度高)
  3. 无历史 → 用默认规则 (作者预设)
  4. 多选项 → 按 count 排序, 最近的选择权重更高
    ↓
输出: play "Stan" (置信度 0.78, 基于 7 次历史)
    ↓
如果人换了歌 → 记录新选择 → 更新画像
```

## 四、冷启动

```
新用户, 没有任何历史:

  第 1 天: 用作者预设
    下雨 → 作者推荐《Stan》→ 用户接受 → 记录

  第 3 天: 已经有 3 个场景, 5 个选择
    开始用个人画像替代作者预设

  第 7 天: 完全个性化
    系统不再用作者预设, 全用自己的学习结果
```

## 五、隐私

```
所有数据留在设备上。
不分析, 不上传, 不共享。
张三的画像永远不会变成李四的推荐。
```
