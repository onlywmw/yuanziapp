# 认识一个人

> **核心**: 原子回应人的需要之前, 系统必须先了解这个人。
> **原则**: 不是收集数据。是观察、记忆、理解。

---

## 一、系统怎么认识你

```
不是:
  填问卷。选标签。授权数据。
  系统拿到一堆数字, 开始"分析"你。

而是:
  第 1 天。你什么都没告诉它。它只是看。
  你走进咖啡厅。坐下。打开降噪。放了爵士。
  它不知道你为什么这么做。它只是记住了。

  第 3 天。同样咖啡厅, 同样降噪, 但今天你没放爵士——你放了播客。
  它注意到: 不一样。但它不知道为什么。只是记住。

  第 10 天。它开始注意到: 周末你来这里, 总是放爵士。工作日放播客。
  它不需要知道"为什么"。它只需要知道"什么时候"。

  第 30 天。它认识你了。不是"分析"了 30 天的数据。
  是和你一起经历了 30 天。它记得你的节奏。
```

## 二、记住什么

```
不是所有数据。是变化。

  你每天喝咖啡 → 不记。这是你。
  你今天没喝咖啡 → 记。"今天不太一样。"

  你总是坐靠窗 → 不记。这是习惯。
  你今天坐了角落 → 记。"为什么换了位置？"

  你每次离开时都切换成走路模式 → 记。"这是你的结束仪式。"

系统不记录你的"平均"。系统记录你的"意外"。
意外才有信息。习惯只是你。
```

## 三、沉默也是了解

```
你不打开 App。你忽略了建议。你什么都没做。
这些都是信号。

  系统推荐了一首快歌。你没切。但你调小了音量。
  → "不是不喜欢快歌。是这个时刻不想太响。"

  系统问"要不要给妹妹打电话？"你划掉了通知。
  → "现在不是时候。但你没说永远不。"

  今天什么推荐都没给。你也没打开 App。
  → "今天你不想被打扰。好的。"
```

## 四、时间的形状

```
不是"你的偏好: 爵士 > 嘻哈"。
是时间的形状:

  工作日早晨:     快节奏, 短, 不停留
  工作日下午:     长, 深的背景音乐
  周末早晨:       安静, 慢, 不推送
  周末下午:       随机的, 开一些新东西
  雨天:          熟悉的, 有叙事感的
  晴天:          什么都可以

  不是标签。是你在时间里的形状。
  同一个人, 不同时间, 不同形状。
```

## 五、实现

```
一个本地的 person 文件:

{
  "rhythms": {
    "weekday_morning":   {"pace": "fast", "music": "short", "push": false},
    "weekday_afternoon": {"pace": "slow", "music": "deep", "push": false},
    "weekend_morning":   {"pace": "calm", "music": "ambient", "push": false},
    "weekend_afternoon": {"pace": "open", "music": "explore", "push": true}
  },
  "exceptions": [
    {"date": "2026-07-15", "expected": "jazz", "actual": "podcast", "note": "工作日但放了播客"}
  ],
  "silences": [
    {"date": "2026-07-18", "action": "lowered_volume", "context": "rainy_cafe"}
  ]
}

不是大模型。就是一个小文件。
每次交互, 更新一点点。
30 天后, 这个文件就是你。
```
