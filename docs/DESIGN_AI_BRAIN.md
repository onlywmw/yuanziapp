# AI 原子 — 不是路由, 是大脑

> **问题**: system.ai 被当成意图分类器。输入→匹配→输出。这不是智能。
> **真正的智能**: 理解为什么、注意到人自己没注意到的事、跨领域联想、看穿情绪

---

## 一、现在 vs 应该

```
现在 (路由器):
  下雨 + 咖啡厅 → 匹配规则 → 播《Stan》
  做了 100 次, 每次都一样。
  准确。无聊。不聪明。

应该 (大脑):
  下雨了。
  但今天的雨和昨天不一样——你今天没打伞。
  你的步数连续三天下降。
  你 10 天没给妹妹打电话了。
  上周这个时候你在咖啡馆笑。今天没有。
  
  不是播歌。
  是问: "要不要给妹妹打个电话？"
  
  这不是规则匹配。这是理解。
```

## 二、大脑做什么

### 理解, 不是匹配

```
规则: rain + cafe → Stan
理解: 雨声让你安静 → 你选择有叙事感的音乐 → Stan 是其中之一
      → 不是"雨=Stan", 是"雨→安静→需要故事"

下次你听 podcast、你看电影、你读文章——
大脑知道: 你需要的不是《Stan》, 是"一个有故事的东西"。
```

### 注意到人自己没注意到的事

```
数据: 连续 4 天听忧郁音乐, 步数下降 60%, 没给妹妹打电话
规则: 无法匹配任何规则 → 什么都不做
大脑: "你最近是不是不太开心？"
     → 不是负面判断, 是关切
     → 不说"你应该开心", 说"我注意到一些变化"
```

### 跨领域联想

```
昨天: 你在咖啡厅写了一首诗
今天: 下雨了
明天: 不知道

大脑:
  "去年这个时候你也写了一首诗。也是雨天。
   你把它发给了妹妹。她回了一句话让你笑了。
   要不要再发一次？"
  
  这不是推荐音乐。
  这是在你生活的碎片中找到一条线。
```

### 理解变化, 不是状态

```
不是: "现在 = 下雨 + 咖啡厅 → 播 X"
是:   "今天和昨天不一样。昨天你笑了。今天没有。为什么？"

大脑关心的不是"现在的状态"。
关心的是"状态怎么变"。
变化里有情绪, 状态里没有。
```

## 三、大脑怎么做到

```
不靠外部 API。不靠大模型。

靠本地数据 + 时间维度:

  1. 行为流: 你做了什么, 按时间排列
     听歌 → 写诗 → 打电话 → 走路 → 沉默
     → 大脑看到的不只是一次行为, 是一段时间里的自己

  2. 变化检测: 不是绝对值, 是变化幅度
     步数 8000 → 5000 → 2000  ← 连续下降, 不只是"今天少了"

  3. 周期对比: 和过去的自己比
     上周三 vs 这周三
     这个雨天 vs 上个雨天
     这个咖啡厅 vs 上次来的时候

  4. 联想: A 和 B 有关系不是规则定的
     听忧郁音乐 + 步数下降 + 没联系妹妹
     → 不是三个独立事件
     → 是同一个模式的不同信号

  5. 克制: 不频繁打扰
     看到模式 → 不马上说
     连续 3 天确认 → 才轻轻问一句
     人可以选择忽略
```

## 四、大脑和规则引擎的关系

```
规则引擎:
  下雨 + 咖啡厅 → 播音乐
  准确, 可靠, 永远工作

大脑:
  观察规则引擎的输出
  观察人的反应 (接受/拒绝/忽略)
  观察跨领域的变化

  规则引擎做"正确"的事。
  大脑偶尔在正确之外, 多说一句。

  100 次交互里:
    95 次: 规则引擎处理, 大脑沉默
    5 次: 大脑觉得时机到了, 轻轻说一句
```

## 五、大脑不说话的时候

```
大脑大部分时间不说话。
它只是在看, 在学习, 在理解。

一个月后:
  它知道你的雨季模式
  它知道你在咖啡厅和家里的区别
  它知道你什么时候需要安静, 什么时候需要被提醒
  它知道妹妹的电话对你很重要

然后某一天:
  下雨了。你没去咖啡厅。你待在家里。
  步数降到 200。第 4 天没给妹妹打电话。

  大脑: "要不要给妹妹打个电话？"

  你说: "好。"

  这是智能。不是推荐。不是规则。不是设定。
  是一个默默看了你一个月的东西, 在合适的时候说了一句对的话。
```
