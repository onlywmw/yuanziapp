# 原子连接规则

> **性质**: 架构约束 — 原子间依赖关系的允许/禁止/警告规则
> **原则**: 基座稳定、安全优先、连接可控

---

## 一、层级规则

```
              ┌─────────────┐
              │ 注册原子     │ ← 可依赖基础原子 + 其他注册原子
              │ (61+ 个)    │
              ├─────────────┤
              │ 基础原子     │ ← 只可依赖基础原子，不可依赖注册原子
              │ (13 个)     │
              └─────────────┘
```

| 从 → 到 | 基础原子 | 注册原子 |
|----------|---------|---------|
| 基础原子 | ✅ 允许 | ❌ 禁止 |
| 注册原子 | ✅ 允许 | ⚠️ 有条件 |

## 二、为什么基础原子不能依赖注册原子

```
假设 file-read 依赖了 mcp.s3:

  用户删除了 mcp.s3
       ↓
  file-read 崩溃
       ↓
  所有依赖 file-read 的原子全部崩溃
       ↓
  整个系统不可用
```

**基础原子是地基。地基不能建在可拆卸的墙上。**

## 三、危险的连接链

系统检测以下连接模式时发出警告，但不强制阻断：

### 3.1 入站 → 写文件

```
http-get ──→ file-write
http-post ──→ file-write

风险: 网络内容直接写入文件系统，可能植入恶意代码
建议: 中间必须经过内容校验
```

### 3.2 读文件 → 出站

```
file-read ──→ http-post
file-read ──→ http-get (URL 参数来自文件内容)

风险: 本地文件内容泄露到外部
建议: 文件内容不得作为 URL 参数
```

### 3.3 解密 → 出站

```
decrypt-aes ──→ http-post
decrypt-aes ──→ file-write (路径来自外部)

风险: 解密后的敏感数据被发送或写入不受控位置
建议: 解密后必须有用户确认或白名单校验
```

### 3.4 外网输入 → 命令执行

```
http-get ──→ math-calc (eval 模式)
http-get ──→ string-match (pattern 来自外部)

风险: 外部输入作为代码/正则执行 (ReDoS, 代码注入)
建议: 正则和表达式禁止来自外部输入
```

### 3.5 循环写入

```
file-read ──→ file-write ──→ file-read ──→ file-write
               ↑_____________________________|

风险: 无限循环读写，磁盘写满
建议: 检测依赖图中的循环路径
```

## 四、正确连接示例

```
✅ 允许:
  file-dir ──→ file-read         (列举后读取)
  http-get ──→ json-parse        (获取后解析)
  string-split ──→ math-calc     (拆分后计算)
  json-parse ──→ hash-digest     (解析后哈希)
  file-read ──→ encrypt-aes      (读取后加密)
  encrypt-aes ──→ file-write     (加密后存储)
  mongodb ──→ http-get           (注册原子依赖基础原子)
  postgres ──→ json-parse ──→ string-split   (链式依赖)

❌ 禁止:
  file-read ──→ mcp.s3           (基础依赖注册)
  encrypt-aes ──→ mcp.postgres   (基础依赖注册)

⚠️ 警告:
  http-get ──→ file-write        (入站→写文件)
  file-read ──→ http-post        (读文件→出站)
  http-get ──→ decrypt-aes       (外网输入→解密)
```

## 五、注册原子间的循环检测

```
注册原子A ──→ 注册原子B ──→ 注册原子C
               ↑               │
               └───────────────┘

系统在 register 时自动运行 resolve_dependencies() 检测循环。
有循环 → 注册被拒，直到解除循环依赖。
```

## 六、运行时连接约束

| 约束 | 说明 |
|------|------|
| 原子不可调用自身 | handler 不可以递归调用自己 |
| 异步超时 | 链式调用总超时 ≤ 30s |
| 深度限制 | 调用链最大深度 5 层 |
| 调用计数 | 每个原子的内存中维护调用次数，防死循环 |

---

> **连接规则是原子生态的交通法规——不是限制自由，是防止撞车。**
