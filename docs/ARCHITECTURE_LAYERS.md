# Yuanzi 三层架构

> **定位**: 基础层定义 → 发现层分发 → 引擎层组合执行
> **原则**: 每层独立, 下层不知道上层, 上层调用下层

---

## 一、三层模型

```
┌─────────────────────────────────────────────┐
│  引擎层 (Graph Engine)                        │
│  · 图谱渲染: 节点/连线/通道/动画              │
│  · 工作流执行: 拓扑排序/通道转换/容错         │
│  · 参数面板: 混音台/推子/配色                 │
│  · 运行时可视化: 数据流过/状态变化             │
│                                              │
│  引擎不关心原子是"自带的"还是"下载的"。        │
│  对引擎来说, 它们都是节点和连线。              │
├─────────────────────────────────────────────┤
│  发现层 (M7)                                  │
│  · 原子市场: 搜索/浏览/评分/安装              │
│  · 工作流市场: 模板分享/一键安装               │
│  · 通道发现: 可复用通道的安装与推荐            │
│  · 作者信誉: 认证/评分/信任                   │
│                                              │
│  发现层是"应用商店"。负责找到和分发。           │
├─────────────────────────────────────────────┤
│  基础层                                       │
│  · 原子体系: 五类25原子/生命周期/I-O/版本/安全 │
│  · 通道体系: 五种通道/映射语法/容错策略/版本  │
│  · 工作流体系: DAG定义/连线验证/执行引擎       │
│  · 注册中心: submit/review/status/probe       │
│                                              │
│  基础层是"发动机"。负责定义和运行。             │
└─────────────────────────────────────────────┘
```

## 二、层间关系

```
基础层 → 发现层:
  · 基础层提供: 原子列表API、通道列表API
  · 发现层消费: 在市场中展示、搜索、安装

发现层 → 引擎层:
  · 发现层提供: 用户已安装的原子和通道
  · 引擎层消费: 渲染为图谱节点和连线

基础层 → 引擎层:
  · 基础层提供: 工作流定义、通道定义、原子元数据
  · 引擎层消费: 执行工作流、渲染通道动画

引擎层不直接调发现层。发现层不修改基础层的定义。
```

## 四、一个原子的旅程

```
张三在基础层创建了一个原子:
  → 定义 atom_id、I/O Schema、soul 字段
  → 注册 → 签名 → 上链

张三发布到发现层:
  → 出现在市场中
  → 李四搜索到 → 查看详情 → 安装

李四安装后, 引擎层接管:
  → 原子出现在李四的图谱上
  → 李四把它接入自己的工作流
  → 引擎执行、渲染、动画

三层各司其职。
张三改了原子 → 基础层更新签名 → 发现层通知李四有新版本 → 引擎层更新图谱。
```
