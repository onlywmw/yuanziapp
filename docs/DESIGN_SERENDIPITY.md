# 灵感机制

> **问题**: 推荐越用越窄。怎么让手机偶尔给你惊喜？
> **方案**: 在安全区内偶尔越界 — 不是乱推, 是有理由的突破

---

## 一、推荐 vs 灵感

```
推荐: "你总是雨天听《Stan》, 给你《Stan》"
      → 安全, 准确, 无聊

灵感: "你总是雨天听《Stan》。但上次周六雨天你选了爵士。
       而且你已经 3 周没碰过 Radiohead 了。
       试试《Creep》？"
      → 有一点冒险, 但每一条都有理由
```

## 二、灵感来源

```
不是在几十万首歌里随机选。

灵感来自你的数据:
  1. 时间维度: 上次做这件事是什么时候? 太久没做的事值得试
  2. 情绪对比: 上次在这个场景你选了完全不同的风格
  3. 时间模式: 同样是雨天, 周六和周三选的不一样
  4. 遗忘的偏好: 以前常听但最近没听的
  5. 相邻偏好: 和你口味相似的人也喜欢什么 (本地, 不联网)

每一条灵感建议都附上理由:
  "上次周六雨天你听了爵士"
  "你已经 3 周没听 Radiohead 了"
  "和《Stan》同一年发行的"

不是"猜你喜欢"。
是"提醒你曾经喜欢过的东西"。
```

## 三、实现

```
决策原子的输出不只是 "play Stan (score=0.78)"。

多一条: "inspiration: Creep (reason: 3周未听, 上次周六雨天选了爵士)"

用户看到:
  ┌──────────────────────────────────────┐
  │  ☔ 下雨了, 为你推荐                  │
  │                                      │
  │  《Stan》          (你最常听的)       │
  │  《Creep》  ✨      (3周没听了)        │
  │  《爵士精选》       (上次周六听的)     │
  │                                      │
  └──────────────────────────────────────┘

系统推荐常用, 灵感推荐一个你曾喜欢但最近忘了的。
你选哪个都行。灵感只是多一个选项。
```

## 四、灵感概率

```
不是每次都推灵感。大部分时间推准确的。

  80%: 推荐 (最常听的, 最高置信度)
  15%: 灵感 (遗忘的/时间相关的/情绪对比的)
  5%:  探索 (完全随机的, 相邻偏好的)

人大部分时间需要准确。
偶尔需要被提醒: 你曾经喜欢过这个。
```

## 五、和推荐的区别

```
推荐引擎:
  学习偏好 → 优化准确率 → 越来越准 → 越来越窄

灵感:
  学习偏好 → 偶尔主动突破 → 提醒人遗忘的东西
  不是"猜你要什么", 是"提醒你曾经有什么"
```
