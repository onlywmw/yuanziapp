# 知识图谱视觉设计方案

> **给设计师**: 这份文档是设计依据。参考 Obsidian 关系图谱的感觉，做出属于 Yuanzi 的视觉语言。
> **参考**: Obsidian Graph View — 力导向布局、节点发光、粒子连线、暗色背景

---

## 一、Obsidian 做对了什么

```
Obsidian 关系图谱的特点:

  1. 暗色背景 — 让节点成为视觉焦点
  2. 力导向布局 — 节点像星系一样自然分布
  3. 连线半透明 — 不抢节点的视觉权重
  4. 节点发光 — 选中节点有光晕扩散
  5. 局部图谱 — 只看一个节点和它的邻居
  6. 搜索高亮 — 匹配节点亮起，其余变暗
  7. 平滑动画 — 节点移动有惯性，像在液体中
  8. 节点大小有语义 — 连接越多越大
```

## 二、Yuanzi 要的不同

```
Obsidian                       Yuanzi
────────                       ──────
笔记节点                        原子节点 (作者是原子属性, 展示在详情面板)
连线无类型                      5 种通道 (直通/映射/转换/合并/分流)
静态图谱                        数据流动画 (动脉红/静脉蓝)
个人知识库                      多人生态 (作者连接世界)
无交互                          可点击/拖拽/删除/注册
紫色调                          陶土 clay + 鼠尾草 sage 调
```

## 三、色彩方案

```
背景:
  主背景     #1A1A1A    深灰黑 (像 Obsidian)
  画布网格   #252525    极淡网格线 (几乎不可见)

节点:
  基础原子   #3A3A3A    暗灰, 稳定, 不引人注意
            边框 #555555 虚线 1dp
  注册原子   按分类:
            Database      #D4A017 (amber)
            Cloud         #B85C38 (clay)
            AI            #C23B22 (rust)
            Security      #E8A090 (rust_light)
            Document      #6B8E6B (sage)
            Web           #8B3A1E (clay_deep)
  CENTER     #FFFFFF     纯白, 最大, 最亮

连线:
  依赖线     #444444     极细, 半透明 30%
  通道线     按类型着色, 比依赖线亮
            直通: #666666
            映射: #B85C38
            转换: #D4A017
            合并: #6B8E6B
            分流: #A3C4A3
  高亮线     #FFFFFF     选中节点相关连线变亮

发光:
  选中节点   外发光 8dp, 节点颜色, 透明度 40%
  搜索匹配   外发光 12dp, amber 色, 透明度 60%
```

## 四、布局算法

### 4.1 力导向

```
节点间力:
  引力: 有连线的节点互相吸引 (距离 = 150dp)
  斥力: 所有节点互相排斥 (最小距离 = 80dp)
  中心力: CENTER 节点有额外引力, 让图谱向心

稳定后:
  节点缓慢微动 (±2dp), 像在呼吸
  新建节点从 CENTER 位置扩散出现
  删除节点淡出消失
```

### 4.2 分层显示

```
缩放级别控制可见内容:

  0.3x - 0.5x:  只显示 CENTER + YUANZI, 其余为光点
  0.5x - 0.8x:  显示注册原子名称 (12sp)
  0.8x - 1.5x:  显示基础原子 + 连线标签
  1.5x - 3.0x:  显示完整详情 (功能数, 评分, 作者)
```

### 4.3 局部图谱模式

```
双击任一节点 → 进入局部图谱:
  · 只显示该节点和它的直接邻居
  · 其余节点淡出 (alpha 10%)
  · 标题栏显示 "mcp.postgres 的关系图谱"
  · 再次双击空白区域 → 返回全局图谱
```

## 五、动效规格

### 5.1 节点状态

```
常态:        稳定, 微动 (±2dp)
悬停:        放大 1.1x, 外发光, 200ms
选中:        放大 1.15x, 强发光, 边框白, 200ms
新增:        CENTER 位置 scale 0→1, 飞到目标位置, 400ms
删除:        缩小 + 淡出, 300ms, 相邻节点重新布局
状态变化:    颜色渐变, 500ms (如 running→unreachable)
```

### 5.2 连线

```
常态:        半透明 30%, 无动画
悬停:        变亮 80%, 显示线类型标注
数据流过:    红色/蓝色光点沿连线匀速移动
            速度: 直通线 3px/帧, 映射线 1.5px/帧
            宽线: 光点更大更亮
```

### 5.3 画布

```
平移:        跟手, 惯性减速
缩放:        双指中心缩放, 有弹性边界
回中:        双击空白 → CENTER 节点回到屏幕中心, 400ms 缓出
```

## 六、场景流程

### 6.1 打开 App → 看到什么

```
1. 暗色背景淡入 (200ms)
2. CENTER 节点出现, 发光 (300ms)
3. YUANZI 节点出现, 连线生长到 CENTER (200ms)
4. 注册原子从 CENTER 扩散到各自位置 (400ms)
5. 基础原子淡入, 在底层 (300ms)
6. 原子详情面板淡入, 显示作者信息 (200ms)

总启动动画: ~1.6 秒
```

### 6.2 搜索 → 看到什么

```
1. 输入搜索词
2. 匹配节点亮起 (amber 发光)
3. 不匹配节点变暗 (alpha 20%)
4. 匹配节点连线高亮 (白色)
5. 第一个匹配节点自动居中 + 放大
6. 清除搜索 → 全部恢复常态
```

### 6.3 原子注册 → 看到什么

```
1. 新节点在 CENTER 位置生成 (scale 0)
2. 缩放到正常大小 (300ms)
3. 沿曲线飞到力导向计算出的位置 (500ms)
4. 相关连线生长 (200ms)
5. 新原子节点的详情面板高亮显示作者署名
```
