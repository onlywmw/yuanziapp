# 活系统 — 原子自组合

> **问题**: 死模板。下雨→工作流→Stan。分支再多也是死模板。
> **活系统**: 每个原子实时发出状态, AI 观察状态云, 发现此刻的组合, 输出永远是新的

---

## 一、状态云

每个原子不是"触发", 是"持续发出状态"。

```
此刻的状态云:

  weather:   雨, 大雨, 下了 2h, 气压 1003hPa, 雨声密度 0.8
  location:  常去咖啡厅, 坐了 45min, 靠窗
  time:      周六, 15:00, 你通常 16:00 走
  device:    耳机降噪开, 电量 72%, 音量 60%
  health:    步数偏少, 心率平稳
  calendar:  接下来 2h 空闲
  music:     这周 80% 爵士, 今天还没听, 上次《Stan》是 4 天前
  camera:    暖光, 有咖啡杯, 窗外有雨痕
  memory:    上次在这里听《Blue in Green》停了 3 秒 ← 你犹豫了
```

每个原子贡献一个维度。这些维度同时在变化。没有触发条件, 只有"此刻是什么"。

## 二、AI 做的事

不是匹配规则。是观察状态云, 找出有意思的组合。

### 第一步: 关联

```
不是 "weather = rain" → 触发音乐。

而是:
  雨声密度 0.8 + 爵士这周听了很多 + 降噪开着 + 上次《Blue in Green》你犹豫了
  → 这几个维度有关联
  → 关联强度: 0.87
```

AI 在每个时刻扫描所有维度, 找到关联最强的一组。

### 第二步: 组合

```
关联网: {weather.雨声密度, music.本周偏好, memory.上次犹豫}

AI 试组合:

  组合 A: 一首像雨声密度的爵士 → 匹配度 0.82
  组合 B: 一个 45 分钟的爵士深挖列表 → 匹配度 0.78  
  组合 C: 一张窗外雨痕视觉效果的专辑 → 匹配度 0.71
  组合 D: 一个雨声音乐混合的现场录音 → 匹配度 0.65

不是预设。是此刻的几个维度, 交叉运算出来的。
```

### 第三步: 输出

```
选组合 A + B:
  输出: "45 分钟爵士深挖。从《Blue in Green》开始——
        上次你停在这里, 这次听听后面。刚好在你走之前结束。"

不是"推荐 3 首歌"。
是根据所有状态, 织出一个完整的体验。
```

## 三、和死模板的区别

```
死模板:
  下雨 → 激活工作流 → 播歌

活系统:
  此刻的状态云有 12 个维度
  AI 扫描: 哪些维度有关联?
    · 雨声密度 + 本周爵士偏好 + 上次犹豫 = 0.87
    · 雨声密度 + 降噪开 + 42 分钟空闲 = 0.74
    · 窗外雨痕 + 暖光 + 咖啡杯 = 0.68 (这是视觉, 不是音乐)
  → 组合: 爵士深挖列表 + 雨声密度匹配 + 45 分钟时长
  → 输出: 一段话 + 一个列表 + 从你上次犹豫的地方开始

  明天同样下雨, 在咖啡厅。
  但明天是周日, 你坐了 3 小时, 窗外是细雨不是大雨, 你上周没听爵士。
  → 状态云完全不同
  → 输出完全不同
```

## 四、组合空间

```
不是作者预设"什么触发什么"。
是 AI 在所有原子之间探索可能的组合。

活跃原子: 12 个
每个原子有 3-5 个维度
总维度: ~50 个
每次扫描: 找关联最强的 3-5 个维度
可能的组合: C(50,5) = 2 百万种

不是暴力算 2 百万种。
而是:
  1. 只扫描活跃维度 (12 个原子 × 平均 4 维度 = 48)
  2. 过滤弱关联 (<0.5)
  3. 保留 top 3 个关联组
  4. 每组生成 3-5 个组合
  5. 按用户历史偏好排序

输出: 1-2 个最佳组合。
```

## 五、越用越活

```
第 1 天:
  状态云有 12 个维度
  AI 随便试几个组合, 你接受了 1 个
  → 记录: 这些维度在这种组合下有效

第 30 天:
  AI 已经见过 200 种状态云
  知道哪些维度经常一起出现
  知道哪些组合你喜欢, 哪些你忽略
  知道什么时候该试新组合, 什么时候该用老的

第 365 天:
  状态云一出现, AI 就知道该怎么组合
  不是预设规则
  是见过足够多, 知道怎么处理
```

## 六、和个性化、灵感的关系

```
个性化:  记住你的偏好
灵感:    偶尔突破偏好
活系统:  不依赖偏好, 也不预设规则——此刻的状态云决定一切

个性化说: "你总是雨天听 Stan" → 推荐 Stan
灵感说:   "试试 Creep, 你 3 周没听了"
活系统说: "雨声密度 + 降噪 + 45 分钟 + 你上次犹豫的那首 
           + 窗外雨痕的光线 + 你明天没有早会
           → 这不是一个音乐推荐, 是一个 45 分钟的体验"
```

## 七、技术实现

```
核心: 向量 + 注意力 + 组合探索

每个原子的状态 → 向量 (128 维)
  weather: [0.8, 0.3, 0.1, ...]  ← 雨声密度, 湿度, 气压...
  music:   [0.2, 0.7, 0.5, ...]  ← 本周偏好, 去年偏好, 上次犹豫...
  ...

状态云 = 堆叠所有向量 → 矩阵 (12 × 128)

关联计算:
  attention(状态云) → 关联矩阵 (12 × 12)
  取关联最强的 top 3 组

组合生成:
  每组关联: 穷举可能的输出形式
    · 时间长度 (5min/45min/2h)
    · 媒体类型 (音乐/视频/播客/安静)
    · 起点 (上次中断处/全新/随机)
    · 强度 (匹配雨声/对比雨声/忽略雨声)
  每组生成 3-5 个组合

排序:
  用户历史偏好 + 今日新颖度
  不排"最准确的", 排"最适合此刻的"
```
