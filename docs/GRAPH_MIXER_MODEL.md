# 图谱视角混合器

> **核心**: 没有"该显示什么"的固定答案。每个人的视角不同——有人想看系统的血管, 有人想看终端的作品。
> **方案**: 一个滑动条, 人自己调到想看的位置。

---

## 一、两个极端

```
工作流视角 (滑块左边)              终端视角 (滑块右边)
─────────────────────              ─────────────────────
看到什么:
  原子之间的连线                     原子本身的样子
  数据在通道里流动                   风格标签、叙事
  运行记录、日志                     评分、评论
  谁依赖谁、谁调用谁                 谁是作者、谁在用

适合:
  开发者、系统管理员                 普通用户、浏览者
  调试、优化、排错                   发现、收藏、分享

感觉:
  像看引擎运转                       像逛市集
  机械的美感                         人文的美感
```

## 三、滑块如何影响图谱

### 3.1 节点可见度

每种原子有两个权重值: 管道权重 和 作品权重

```
                管道权重  作品权重
基础原子 (system.*)    1.0      0.0     ← 只在管道视角出现
基础设施 (mcp.*)       0.7      0.3     ← 偏管道
半成品 (有功能无soul)   0.4      0.6     ← 偏作品
终端原子 (有soul)       0.2      1.0     ← 偏作品
精选 (高分+叙事)        0.1      1.0     ← 纯粹的作品

系统根据原始权重预先分类, 不需要作者手动设置。
```

### 3.2 滑块位置 → 实际可见度

```
滑块位置: 0.0 (最左, 纯管道) ~ 1.0 (最右, 纯作品)

节点可见度 = 管道权重 × (1 - 滑块) + 作品权重 × 滑块

示例:

滑块=0.0 (纯管道):
  system.* 可见度 = 1.0 × 1.0 + 0.0 × 0.0 = 1.0 → 完全可见
  终端原子 可见度 = 0.2 × 1.0 + 1.0 × 0.0 = 0.2 → 几乎不可见

滑块=0.5 (中间):
  system.* 可见度 = 0.5
  终端原子 可见度 = 0.6

滑块=1.0 (纯作品):
  system.* 可见度 = 0.0 → 完全隐藏
  终端原子 可见度 = 1.0 → 完全可见
```

### 3.3 渲染

```
可见度 0.0~0.2:  不渲染节点
可见度 0.2~0.5:  渲染为小光点, 无标签
可见度 0.5~0.8:  渲染缩略节点, 有标签
可见度 0.8~1.0:  渲染完整节点, 标签+详情摘要
```

## 四、通道和连线的变化

```
管道视角 (滑块左边):
  连线粗, 动画明显
  数据流动画全速 (红蓝光点)
  线型区分明显 (直通/映射/转换/合并/分流)
  连线标签显示
  → 看到的是"系统怎么运转"

作品视角 (滑块右边):
  连线细, 半透明
  数据流动画关闭或极慢
  只保留直接依赖线, 隐藏通道细节
  连线标签隐藏
  → 看到的是"有什么好东西"
```

```
[💎 发现]    滑块=1.0   纯作品, 看有什么好东西
```

## 六、原子权重如何确定

不需要作者手动设置。系统根据已有数据自动计算:

```python
def compute_weights(atom):
    has_soul = bool(atom.classification.soul?.narrative)
    has_ratings = atom.ratings and atom.ratings.count >= 10
    is_system = atom.atom_id.startswith("system.")
    is_mcp = atom.atom_id.startswith("mcp.")

    if is_system:
        return (1.0, 0.0)      # 纯管道
    if has_soul and has_ratings:
        return (0.1, 1.0)      # 纯作品
    if has_soul:
        return (0.2, 0.9)      # 偏作品
    if is_mcp:
        return (0.7, 0.3)      # 偏管道
    
    return (0.5, 0.5)          # 默认均衡
```

## 八、总结

```
旧思路: 系统决定"哪些原子值得展示"
新思路: 人自己决定"我现在想看什么"

工作流视角 → 看到系统的血管和脉搏
终端视角 → 看到人的作品和灵魂

不是分类, 是视角。
不是标签, 是偏好。
不是规则, 是感觉。
```

---

> **图谱不是一张静态地图。它是一个活的万花筒——转一下, 看到的是管道; 再转一下, 看到的是作品。人在中间, 自己调。**
