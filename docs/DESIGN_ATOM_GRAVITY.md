# 星云引擎 (Nebula Engine)

> **定位**: 原子不需要中央大脑。系统先认识人, 原子再回应人的需要。
> **原则**: 先有人的档案, 再有原子的共振。共振不是原子对原子, 是原子对人的回应。

---

## 一、先认识人

系统做的第一件事不是分析数据——是和你一起生活。

```
第 1 天: 什么都不了解。只是看。
第 7 天: 有了节奏的轮廓。知道什么时候不该打扰。
第 30 天: 认识你了。知道你的形状、沉默、收藏。
```

人的档案 (DESIGN_PERSON_ARCHIVE.md):
- 节奏: 你的时间形状
- 回应: 什么让你停下, 什么让你划过
- 沉默: 你没说的那些
- 连接: 你通过原子连接的人
- 成长: 你变了多少

## 二、原子回应人

不是原子和原子共振。是每个原子问同一个问题: "这个人此刻需要什么？"

```
你刚进来, 冷, 想安静, 不赶时间。

weather:  雨声 → 安静需求      回应 0.9
music:    慢爵士 → 安静+过渡    回应 0.85
coffee:   暖 → 冷需求           回应 0.95
device:   降噪 → 不被干扰       回应 0.9
calendar: 空闲 → 不赶            回应 0.8

回应相近的原子自然聚在一起。
不是 AI 算的。是它们都在回答同一个问题。
```

## 三、共振维度

每个原子发出 5 类维度信号, 交叉计算 (DESIGN_RESONANCE_DIMENSIONS.md):

- 物理: 强度·密度·节奏·变化率
- 时间: 时长·周期·持续性·转折点
- 情绪: 能量·开放度·熟悉度·新鲜度
- 状态: 专注度·移动性·社交性·舒适度
- 关系: 互补·因果·对比·协同

共振实现 (DESIGN_RESONANCE_IMPLEMENTATION.md):
- 同类别 ×2, 跨类别 ×0.5
- 权重学习: 用户接受 +0.05, 忽略 -0.03
- 12 原子 × 5 维度 × 66 对 < 1ms

## 四、不是模板

```
模板:
  下雨 → 激活咖啡厅工作流 → 播《Stan》

引力:
  此刻的标签云是 [rain, jazz, weekend, headphones, 45min, window]
  3 周前同样下雨, 但那天是 [rain, hiphop, monday, speaker, 10min, desk]
  → 标签完全不同 → 共振结果不同 → 输出不同

同一个场景, 每次输出都不一样。
因为标签云不一样。
因为原子不是匹配规则——是共振。
```

## 六、个性化

```
不需要单独的"个性化引擎"。

用户接受了一个集群输出 → 参与原子的标签权重 +1
用户忽略了一个集群输出 → 参与原子的标签权重 -0.5
用户改了输出 → 标签做更大的调整

30 天后:
  天气原子知道: 雨天的 melancholy 标签对这个人特别重要
  音乐原子知道: 这个人的 jazz 偏好比 hiphop 强 3 倍

原子自己学会的。不需要中央模型。
```

## 七、灵感

```
不是单独的"灵感系统"。

80% 时间: 用共振最强的组合 (准确)
15% 时间: 随机选一个共振不是最强但有重叠的原子加入集群
           → 给集群带来新鲜感
5% 时间:  完全随机的原子加入
           → 碰运气

不需要专门的灵感算法。
偶尔改变一下集群的成员, 自然就有了意外。
```

## 八、共振的深度

标签重叠不够。同一个标签, 程度不同:

```
weather:  [rain, intensity=0.8, duration=2h]
music:    [jazz, tempo=slow, complexity=high]

rain × jazz → 没有共同标签
但 rain.intensity(0.8) 和 jazz.tempo(slow) → 共振
大雨的密度和慢爵士的节奏 → 不只是标签, 是维度的匹配
```

### 维度共振

```
每个标签带值, 不只是有/没有:

weather 的场:
  rain:          0.8   ← 大雨, 不是小雨
  melancholic:   0.6   ← 有点, 不是极度
  duration:      2.0   ← 持续 2 小时了

music 的场:
  jazz:          0.9   ← 很爵士
  slow:          0.7   ← 慢节奏
  complex:       0.8   ← 丰富层次

calendar 的场:
  free:          1.0   ← 完全空闲
  weekend:       1.0   ← 是周末

共振计算:
  weather.rain(0.8) × music.slow(0.7) = 0.56    ← 雨声密度和慢节奏共振
  weather.melancholic(0.6) × music.jazz(0.9) = 0.54  ← 忧郁和爵士共振
  weather.duration(2.0) × calendar.free(1.0) = 2.0   ← 时间和空闲共振最强

不是标签匹配。是维度值之间的乘积。
时间维度主导了这次共振——2 小时空闲是最强的信号。
```

### 原子自带维度权重

```
每个原子知道自己的哪些维度对哪些原子重要:

weather 知道:
  对 music:   rain 强度最重要, melancholic 次之
  对 calendar: duration 最重要
  对 camera:  不重要 (不共振)

这个权重是学出来的, 不是人定的。
30 天后, weather 自己学会了: 
  "这个人在意雨的强度, 不在意湿度"
```

## 九、演进路线

```
Phase 1 (现在): 维度共振
  原子输出标签 + 值: {rain: 0.8, melancholic: 0.6, duration: 2h}
  共振 = 维度值的加权乘积
  每个原子自带维度权重 (初始均匀, 逐渐学习)

Phase 2 (最终): 向量引力
  原子自己学习状态 → 128 维向量
  不再有"rain"、"melancholic"这些标签
  128 个维度是原子自己对此刻的理解
  
  真正的物理 AI — 原子自己感知、自己理解、自己共振
```

## 九、实现

```
每个原子加一个 gravity 层, 不改 handler:

  gravity.tags(state)      → [rain, melancholy, afternoon]
  gravity.resonate(others) → 标签重叠 → 共振强度
  gravity.join(cluster)    → 贡献自己的部分

  计算量: 12 个原子 × 几个标签 × 66 对 = < 1ms
  学习:   用户的选择就是训练信号
  中心:   没有。原子自己算自己的。
```
