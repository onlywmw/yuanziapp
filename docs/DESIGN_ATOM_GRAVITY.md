# 原子引力系统

> **定位**: 原子不需要中央大脑。每个原子有场, 场之间自然共振, 共振强的聚成集群, 集群就是此刻的输出。
> **原则**: 不需要 AI 分析、不需要预设工作流、不需要人触发。原子自己活。

---

## 一、为什么不需要大脑

```
大脑模式 (错):
  所有原子上报 → 中央 AI 分析 → 做决策 → 分发指令
  → 规则越来越多, 越来越像死模板

引力模式 (对):
  原子发出场 → 场之间自然共振 → 共振强的聚在一起 → 集群自己形成输出
  → 没有规则, 没有模板, 每次都不一样
```

## 二、每个原子的场

```
原子不只是被调用——它持续发出状态标签:

  weather:  [rain, heavy, 2hours, melancholy]
  music:    [jazz, melancholy, calm, focus]
  calendar: [free, 2hours, weekend]
  device:   [headphones, noise-cancelling]
  location: [cafe, window-seat, 45min]
```

## 三、共振

```
两个原子的场有多少重叠 → 共振强度:

  weather + music:    rain, melancholy → 2 个重叠 → 共振 2
  weather + calendar: 0 个重叠 → 共振 0
  weather + location: 0 个重叠 → 共振 0

只有 music 和 weather 在共振。它俩自然连在一起。
不需要谁告诉它们"天气要和音乐关联"。
标签相同就是关联。
```

## 四、集群

```
共振强的原子互相确认 → 形成集群:

  weather + music + calendar + device
    weather: 贡献声音密度, 情绪基调
    music:   贡献歌单, 风格
    calendar: 贡献时间长度
    device:  贡献输出方式, 音量

  不需要协调者 — 每个原子知道自己的角色。
  集群的形状就是输出:
    一首 2 小时的爵士列表, 匹配雨声密度, 降噪模式下播放
```

## 五、不是模板

```
模板:
  下雨 → 激活咖啡厅工作流 → 播《Stan》

引力:
  此刻的标签云是 [rain, jazz, weekend, headphones, 45min, window]
  3 周前同样下雨, 但那天是 [rain, hiphop, monday, speaker, 10min, desk]
  → 标签完全不同 → 共振结果不同 → 输出不同

同一个场景, 每次输出都不一样。
因为标签云不一样。
因为原子不是匹配规则——是共振。
```

## 六、个性化

```
不需要单独的"个性化引擎"。

用户接受了一个集群输出 → 参与原子的标签权重 +1
用户忽略了一个集群输出 → 参与原子的标签权重 -0.5
用户改了输出 → 标签做更大的调整

30 天后:
  天气原子知道: 雨天的 melancholy 标签对这个人特别重要
  音乐原子知道: 这个人的 jazz 偏好比 hiphop 强 3 倍

原子自己学会的。不需要中央模型。
```

## 七、灵感

```
不是单独的"灵感系统"。

80% 时间: 用共振最强的组合 (准确)
15% 时间: 随机选一个共振不是最强但有重叠的原子加入集群
           → 给集群带来新鲜感
5% 时间:  完全随机的原子加入
           → 碰运气

不需要专门的灵感算法。
偶尔改变一下集群的成员, 自然就有了意外。
```

## 八、演进路线

```
Phase 1 (现在): 标签共振
  原子输出标签 [rain, melancholy, afternoon]
  共振 = 标签重叠数
  简单, 直接, 先跑起来

Phase 2 (最终): 向量引力
  原子自己学习状态 → 128 维向量
  共振 = 余弦相似度
  不是人类定义的标签, 是原子自己的感知

  128 个维度不是"雨"、"湿度"、"气压"
  是原子自己对"这个雨"的理解
  和 3 周前那个雨天的向量对比
  和那天听到的音乐的向量对比
  
  真正的物理 AI — 原子自己感知、自己理解、自己共振
```

## 九、实现

```
每个原子加一个 gravity 层, 不改 handler:

  gravity.tags(state)      → [rain, melancholy, afternoon]
  gravity.resonate(others) → 标签重叠 → 共振强度
  gravity.join(cluster)    → 贡献自己的部分

  计算量: 12 个原子 × 几个标签 × 66 对 = < 1ms
  学习:   用户的选择就是训练信号
  中心:   没有。原子自己算自己的。
```
