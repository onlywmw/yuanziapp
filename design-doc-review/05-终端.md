# 终端组设计文档精读审计报告

## 组概述

覆盖 4 份终端侧文档：APK 架构、Chaquopy 迁移、前端 UI 规格、APK 构建指南。
总体判断：Chaquopy 迁移已在代码中实质落地（`PythonBridge.kt` + `app/build.gradle`），但三份设计文档仍停留在 Termux 时代口径；UI 规格中的通道渲染、市场页、工作流画布在代码中基本未实现；构建指南描述的 aapt2 手工链路已无法产出当前 APK。
对照代码：`widgetmcp_src/`（Kotlin + 内嵌 Python）。

## DESIGN_APK_ARCHITECTURE.md

**一句话定位**：APK 作为唯一人机界面的总体架构与模块清单（ViewFlipper 三页 + 图谱渲染 + REST 轮询）。

**核心决策清单**
- APK 不含业务逻辑，全部经 localhost HTTP 调后端（:8081 REST / :8080 MCP）。
- ViewFlipper 三页：知识图谱首页 / DeepSeek 配置 / Yuanzi 设置。
- 节点 7 类型规格（CENTER/YUANZI/BROWSER/BASE/REGISTERED/ADD_TEMPLATE/AUTHOR_NODE 菱形）。
- 5 种通道线（直通/映射/转换/合并/分流）+ 动脉/静脉/阻断动画。
- 启动握手 `GET :8081/health`，主版本不匹配提示升级；轮询默认 30 分钟。

**关键契约或接口要点**
- `fetchGraph() → GET :8081/atoms?status=registered`；`search → POST :8081/search`；工作流执行 `POST :8081/workflows/{id}/run`（SSE）。
- `/health` 返回 `{version, schema_version}` 用于兼容检查。

**文档与代码现状差距**
- 三页结构与文档一致（`MainActivity.kt:38-42` homeView/buildDeepSeekConfig/buildYuanziSettings）。
- `fetchGraph` 实际打 `:8080/graph`（`yuanzi/YuanziApi.kt:64`），非文档的 `:8081/atoms?status=registered`；`/graph` 由 Yuanzi Core（`yuanzi-atoms/core/main.py:160`，端口 8080）提供，不属注册中心 FastAPI。
- 搜索实际为 `GET /search`（`YuanziApi.kt:86-88`），文档写 POST（内嵌 `api.py` 同时提供 GET/POST，231/241 行，影响小）。
- 节点枚举无 BASE/REGISTERED/AUTHOR_NODE，实际为 CENTER/WIDGET/ADD_TEMPLATE/YUANZI/BROWSER/SETTINGS（`ui/GraphNode.kt:20-27`）。
- 通道渲染未实现：`ui/GraphEdge.kt` 仅 color/width 两字段，全仓库 Kotlin 无 MAP/TRANSFORM/MERGE/SPLIT 任何痕迹（Grep 无匹配）。
- 版本兼容检查未实现：Kotlin 侧无任何读取 `version`/`schema_version` 的代码（Grep 无匹配）。
- §7 模块清单过时：缺 `PythonBridge.kt`、`YuanziSyncService.kt`、`DeepSeekService.kt`、`graph/`（engine/templates/ui）、`domain/`、`data/` 等；`WidgetController` 实际在 `domain/usecase/` 而非"数据层"。

## DESIGN_CHAQUOPY_MIGRATION.md

**一句话定位**：去 Termux 化方案——Chaquopy 内嵌 Python，API 通信由 localhost HTTP 变进程内（建议仍走 HTTP，Kotlin 零改动）。

**核心决策清单**
- Python 代码一行不改；仅改 `DB_PATH`（env `YUANZI_DB_PATH` 兜底 filesDir）。
- `src/main/python/` 存放 registry.py/api.py/migrations/yuanzi-atoms；Chaquopy 16.0.0 插件 + pip 安装 fastapi/uvicorn/pydantic。
- 启动流程：APK 启动 → Chaquopy 初始化 → `api.start_server(filesDir)` → :8081 立即可达。
- 预计 APK 增量 ~25MB、总工作量 ~2 小时。

**关键契约或接口要点**
- Kotlin 入口：`Python.getInstance().getModule("api").callAttr("start_server", filesDir)`。
- DB 新路径 `/data/data/com.nous.widgetmcp/files/agent.db`。

**文档与代码现状差距**
- 方案已落地：`app/build.gradle` 声明 `com.chaquo.python` 16.0.0 且 sourceSets 指向 `../java`、`../res`；`yuanzi/PythonBridge.kt` 实现 ensureStarted→`api.start_server(filesDir)`，`MainActivity.kt:47` 启动时调用；`api.py:432` `start_server(port=8081)` 与文档一致。
- DB_PATH env 模式已实现（`register_mcp_atoms.py:36-38`、`generate_registry_ledger.py:23-25`）。
- pip 依赖超文档清单：实际多装 `requests/jsonschema/pyyaml/cryptography`（`app/build.gradle:23-31`）；其中 cryptography 是 C 扩展包，文档"全部纯 Python、零 C 扩展"结论不再成立（Chaquopy 官方有预编译包可跑，但表述失实）。
- migrations 实际 10 个（001–010），文档写 8 个。
- 文档自身不一致：§3.3 称 DB_PATH"只有一处硬编码要改"，§七步骤 3 又写"修改 DB_PATH (2 个文件)"。
- 文档未提 ABI 裁剪：`build.gradle:34` 已加 `abiFilters arm64-v8a/armeabi-v7a`，直接影响 §六体积估算的准确性。

## DESIGN_FRONTEND_UI.md

**一句话定位**：视觉标准——色彩/字体/间距系统 + 图谱首页、详情浮层、市场页、工作流画布的布局与动效规格。

**核心决策清单**
- clay/sage/rust/amber + 中性色板；标题 22sp/正文 15sp；按钮高 52dp、页边距 20dp。
- 图谱节点 5 类样式；作者信息只进详情面板，"非独立节点"。
- 市场页（搜索 + 4 Tab + 评分卡片）；工作流画布（拖拽连线、运行态边框动画）。
- 动效仅在数据流动时出现；节点出现 300ms、连线生长 200ms、颜色渐变 500ms。

**关键契约或接口要点**
- 色值表（clay #B85C38、sage #6B8E6B、rust #C23B22、amber #D4A017、paper #FAFAFA 等）。
- 详情浮层字段：评分/作者/License/功能列表/状态/版本/签名。

**文档与代码现状差距**
- 色板漂移：`res/values/colors.xml` 仅 clay #B85C38 与文档一致；sage 实为 #7A8B6F（文档 #6B8E6B）、rust #B4453C（#C23B22）、amber #C98A2D（#D4A017）、paper #F7F4EE（#FAFAFA）、ink #2E2A26（#2C2C2C）——代码已自成"Warm Paper"色板。
- 市场页、工作流画布在 APK 中不存在：`MainActivity` 仅三页（首页/DeepSeek/Yuanzi 设置），无任何市场或 DAG 编辑代码。
- 连线样式仅有基础实线：`ui/GraphEdge.kt` 无类型字段，虚线/波浪/箭头/数据流动画均未实现（Grep 无 dash/wave 匹配）。
- 作者节点：本文档与 APK_ARCHITECTURE 矛盾（见跨文档节）；代码两侧都没实现。
- 字体/间距系统（52dp 按钮等）在代码中无统一常量，MainActivity 内联魔法数（如 `tv("…", 22f, …)`）。

## APK_BUILD_GUIDE.md

**一句话定位**：Widget MCP APK 的构建手册——手工依赖、build.sh 命令、端口约定与 6 个坑。

**核心决策清单**
- 三个手动准备文件：android.jar、kotlin-stdlib-1.9.20.jar、nanohttpd-2.3.1.jar。
- 构建命令 `sh build.sh` / `sh build_adb.sh`。
- 端口表：8080=APK 内嵌 McpServer，8081=FastAPI；`usesCleartextTraffic="true"`。
- 6 坑：atoms VIEW 只读、基础原子不可删、作者字段必显、端口冲突、Termux 被杀重试、基础原子随 APK 升级。

**关键契约或接口要点**
- 版本绑定表：APK v2.0 需要 `/api/v1/workflows`、`/api/v1/market`、`/atoms?type=base`。
- 签名一致性要求（debug/release 数据不互通）。

**文档与代码现状差距**
- 构建链路已过期：`build.sh` 是 proot+ssh(8022) 的 Termux 远端 aapt2/d8 流水线，硬编码 `SRC=/root/widget-mcp/app/src/main`，与仓库布局（`widgetmcp_src/{AndroidManifest.xml,java,res}`）不符；且 tar 同步清单只含 domain/data/widget/yuanzi/根 kt，**漏掉 ui/、browser/、graph/ 三个包**，而 AndroidManifest 引用了 `.browser.BrowserActivity`——按此脚本构建必然缺类。
- 两个脚本均不含 Chaquopy/Python 打包，无法产出当前内嵌 Python 的 APK；真正的构建入口是 Gradle（`settings.gradle` + `app/build.gradle`），但仓库无 gradlew，指南全文未提 gradle/Chaquopy。
- 端口表与代码冲突：McpServer 实际默认 **8766**（`McpService.kt:45` `prefs.getInt("mcp_port", 8766)`），8080 实际是 Yuanzi Core（`yuanzi-atoms/core/main.py:17`）；`usesCleartextTraffic="true"` 已正确声明（`AndroidManifest.xml:27`）。
- "基础原子列表硬编码在 APK 中"与现状不符：Kotlin 全库无 `file-read`/BASE_ATOM 硬编码（Grep 无匹配）。
- `libs/` 目录在仓库中不存在，kotlin-stdlib/nanohttpd 缺失——指南虽声明需手动准备，但 Chaquopy 化后该清单已不完整（还缺 Chaquopy 运行时获取方式）。

## 组内跨文档矛盾与不一致

1. **作者节点形态矛盾**：APK_ARCHITECTURE §3.1 规定 AUTHOR_NODE 为独立菱形金色节点；FRONTEND_UI §2.2 明确"作者信息展示在详情面板，非独立节点"。两份文档直接冲突，代码未实现任一方案。
2. **MCP 端口归属矛盾**：APK_ARCHITECTURE 与 BUILD_GUIDE 均称 8080=APK 内嵌 McpServer；代码中 8080=Yuanzi Core（Python），APK McpServer 默认 8766。
3. **图谱数据源矛盾**：APK_ARCHITECTURE §4.1 说 `GET :8081/atoms?status=registered`；代码用 `:8080/graph`；BUILD_GUIDE §三又说图谱数据走 `:8081/stats`——三处三种说法。
4. **搜索方法矛盾**：APK_ARCHITECTURE 写 POST /search；Kotlin 客户端用 GET；后端两者都支持，文档未同步。
5. **Chaquopy 文档内部矛盾**：DB_PATH "一处硬编码"（§3.3）vs "2 个文件"（§七）；migrations 8 个 vs 实际 10 个。
6. **构建指南与迁移方案脱节**：BUILD_GUIDE 全程 Termux/aapt2 口径，未提及 CHAQUOPY_MIGRATION 已确立的 Gradle 构建路径，两份文档给出的"构建方式"互斥。

## 风险与建议（按优先级）

1. **P0 构建指南重写**：build.sh 同步清单漏 ui/browser/graph 且不支持 Chaquopy，按现指南必然构建失败——改为以 Gradle 为唯一入口并补 gradlew。
2. **P0 端口契约统一**：三份文档三种端口说法 vs 代码 8766/8080/8081 三个实际值，且 YuanziApi 单 baseUrl 混调 Core(:8080) 与注册中心(:8081) 两端点（`/search` 在 8081、`/graph` 在 8080），配错端口即 404——需在代码侧拆分 client 或在文档侧固化端口矩阵。
3. **P1 YuanziConfig 默认端口过时**：Chaquopy 落地后注册中心就在本机 8081，`YuanziConfig.kt:31` 默认仍 8080 且注释写"Termux 侧"，嵌入式场景默认配置直连失败。
4. **P1 通道渲染完全缺失**：5 种连线 + 动效是两份文档的核心卖点，Kotlin 零实现——建议先在 GraphEdge 加 type 字段再谈动画。
5. **P1 后端端点缺口**：客户端依赖的 `/agent/widgets` 无任何后端实现（Core 仅有 /graph、/agent/event、/agent/command*）；`/search` 不在 Core 上。
6. **P2 版本兼容检查未实现**：两份文档均要求 /health 主版本检查，代码缺失，升级期会出现静默不兼容。
7. **P2 作者节点决策待定**：先裁决"菱形节点 vs 详情面板字段"，再补实现。
8. **P3 色板对齐**：colors.xml 与 UI 规格 6 个色值不一致，需裁定以哪边为准并单向同步。
9. **P3 Chaquopy 文档刷新**：migrations 数量、pip 清单（含 cryptography 非纯 Python）、ABI 裁剪三处与 build.gradle 现状对齐。
