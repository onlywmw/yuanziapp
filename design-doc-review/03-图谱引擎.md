## DESIGN_GRAPH_ENGINE.md

**定位**：可商用图谱渲染 SDK 的三层架构蓝图（Engine/UI/Interaction，~7000-10000 行，4 期实施）。

**核心决策**
- 三层单向调用，Engine 层零 UI 依赖；Store 单一数据源，Renderer 只读
- 8 模块：Store / NodeManager / EdgeManager / ForceLayout(Barnes-Hut) / Renderer / Camera / Interaction / Animation / Virtualization
- 性能基线：100 节点 60fps、500 节点 30fps、LOD 四档（0.5/1.0/2.0x）、视口 ±20% 裁剪
- `interface GraphEngine` 19 个方法的外部契约（loadGraph/focusNode/enterLocalGraph/animateDataFlow 等）

**关键契约**：GraphEdge 带 5 种通道类型（直通/映射/转换/合并/分流）；Camera zoom 0.3~3.0 + 惯性；Undo/Redo 50 步 Command 栈。

**文档与代码差距**（证据：`widgetmcp_src/java/com/nous/widgetmcp/`）
- 8 模块文件无一存在；实际是 `ui/GraphView.kt`（604 行单体）+ `graph/` 模板钩子层，无 `GraphEngine` 接口
- ForceLayout：`GraphView.kt:249-307` tickPhysics 为 O(n²) 暴力 + 硬编码常数（斥力 3000f、弹簧 0.02f、向心 0.003f、阻尼 0.85），非 Barnes-Hut，无稳定态/呼吸微动
- Camera：`GraphView.kt:533-564` 自承"无视口变换，仅记录状态"；zoom/center 从不应用到 canvas，且无任何调用方（grep 全仓无 zoomTo/centerOn 调用点）
- UI 层（SearchBar/DetailPanel/MiniMap/Toolbar/ContextMenu）与 Interaction 层（LocalGraph/WorkflowMode/UndoRedo/Clipboard）全部缺失
- 5 种通道类型无载体：`GraphEdge.kt` 仅 color/width 两字段，无 ChannelType
- LOD/虚拟化：仅 ObsidianTemplate 有单档 deep-space（<0.3x 光点），无视口裁剪

## DESIGN_GRAPH_REFERENCE.md

**定位**：面向设计师的视觉规范——Obsidian 感觉 + 陶土 clay/鼠尾草 sage 的 Yuanzi 视觉语言。

**核心决策**
- 背景 #1A1A1A、网格 #252525；注册原子按 6 分类着色（amber/clay/rust/rust_light/sage/clay_deep）
- 节点大小有语义（连接越多越大）；选中发光 8dp/40%、搜索匹配 12dp/60% amber
- 悬停 1.1x、选中 1.15x、新增 scale 0→1 飞入 400ms、删除淡出 300ms
- 分层显示阈值 0.3/0.5/0.8/1.5x；局部图谱其余节点淡出 alpha 10%；启动动画 ~1.6s

**文档与代码差距**
- 默认渲染是浅色背景 #F7F4EE（`GraphView.kt:327`、`DefaultTemplate.kt:47`），与暗色规范完全相反
- ObsidianTemplate 背景 #0D0D0D（`ObsidianTemplate.kt:428`）≠ 规范 #1A1A1A（注释自承"更深"，源自 M8 文档）
- clay/sage 色板未进代码：`ObsidianTemplate.kt:484-492` 三个调色板为 Material 蓝/紫/青系（0xFF8AB4F8 等）
- 节点分类着色无数据基础：`GraphNode.kt:20-27` NodeType 为 CENTER/WIDGET/ADD_TEMPLATE/YUANZI/BROWSER/SETTINGS，无 BASE/REGISTERED 及 6 分类
- 已实现且符合：连接数→大小（8dp+2dp/连接）、搜索不匹配变暗 20%、amber 光环、CENTER 呼吸光晕
- 偏离：hover 1.15x（规范 1.1x）；选中光晕 12dp/60%（规范 8dp/40%，12/60 本是搜索匹配档）
- 未实现：启动动画序列、局部图谱淡出（onFocusEnter 空实现，`ObsidianTemplate.kt:232`）、节点新增/删除动画（onNodeAppear/Disappear 安全默认）

## GRAPH_MIXER_MODEL.md

**定位**：管道◄►作品无级滑块，用"视角混合"取代"系统决定显示什么"。

**核心决策**
- 双权重模型：每原子（管道权重， 作品权重），系统自动分类（system.*→(1,0)、有 soul→(0.2,0.9~1.0)、mcp.*→(0.7,0.3)）
- 可见度 = 管道权重×(1-滑块) + 作品权重×滑块；渲染阈值 0.2/0.5/0.8（不渲染/光点/缩略/完整）
- 滑块同时调制连线（粗细/动画/标签）；4 预设（调试 0.0/工作 0.3/浏览 0.7/发现 1.0）
- 偏好存 SharedPreferences，默认 0.5，首启动画引导

**文档与代码差距**
- **核心机制整体未实现**：`mixerPosition` 仅在 `ParameterPanel.kt` 内存取与持久化（encode/decode，行 574/598），全仓无任何消费者；`compute_weights` 不存在（grep 无）；GraphNode 无权重字段；可见度公式与三档渲染阈值无任何代码
- 滑块对连线/动画的调制（第四节）未实现
- 已落地：混音台滑块 UI（`ParameterPanel.kt:406-420`）、4 预设按钮数值与文档一致（work=0.3/browse=0.7/discover=1.0，行 67-84）、SharedPreferences 存储
- 文档瑕疵：§六 `compute_weights` 用 Python 示例，与 Kotlin/Android 实现栈不符

## GRAPH_PARAMETER_PANEL.md

**定位**：8 推子 + 4 配色 + 预设的"调音台"参数面板规格，混音台为主控。

**核心决策**
- 8 参数及范围：节点大小 0.5-2.0x、文字透明度 0-100%、连线粗细 0.5-5dp、连线长度 80-300dp、向心/排斥/吸引 0-1、布局速度慢-快
- 4 配色方案（路径/标签/关键词/属性）；切换 500ms 渐变 + 右上角图例
- 混音台主控：滑块位置映射整组参数（§四给出 0.0/0.5/1.0 三档全参数表）
- 5 内置预设 + 最多 5 个自定义预设；SharedPreferences 独立键（graph.node_size 等）

**文档与代码差距**（`graph/ui/ParameterPanel.kt`，617 行，UI 层高符合）
- 已实现对齐：8 滑块范围逐一相符（行 298-313）；4 配色按钮；5 预设数值与 §四/§五一致（debug 全参数、discover 全参数）；预设 320ms ValueAnimator 平滑过渡；SharedPreferences 持久化与启动恢复
- **6/8 滑块空转**：`GraphView.kt` tickPhysics 硬编码力学常数，从不读 hooks.params——连线长度/向心力/排斥力/吸引力/布局速度 + mixerPosition 拖了无任何效果；仅 nodeBaseSize/textOpacity/edgeThickness/colorScheme 被 ObsidianTemplate 渲染消费（4/8）
- 未实现：自定义预设保存、[↩ 重置默认] 按钮（以🎛默认预设部分代替）、右上角颜色图例、配色 500ms 渐变（代码离散立即切换，行 495-496 注释自承）
- 偏离：存储为单键 "graph_params" 序列化字符串，非文档的 10 个独立键（功能等价）
- 面板无任何宿主接线：grep 全仓仅自身文件引用，MainActivity 未使用

## 组内跨文档矛盾与不一致

1. **数据流粒子速度两套数值**：ENGINE §2.9（动脉 2px/帧、静脉 1px/帧）vs REFERENCE §5.2（直通 3px/帧、映射 1.5px/帧），分类维度与数值均不一致
2. **LOD 分层阈值不一致**：ENGINE §2.10（0.5/1.0/2.0x 四档）vs REFERENCE §4.2（0.3/0.5/0.8/1.5x 四档）vs 代码（<0.3x 单档 + 网格 >0.8x），三方互不相同
3. **混音台坐标系表述相反**：MIXER/PANEL §四为"滑块 0=纯管道"，PANEL §五预设表写"调试 管道=1.0"——数值互补自洽但需读者自行换算，极易误读（代码按滑块值实现，debug=0.0，正确）
4. **节点类型计数三套**：ENGINE 5 种（CENTER/BASE/REGISTERED/YUANZI/BROWSER）vs REFERENCE 交付物"7 种节点" vs 代码 6 种 NodeType
5. **背景色规范漂移**：ENGINE/REFERENCE 均为 #1A1A1A，M8 实施文档改 #0D0D0D 且代码从之，原文档未同步
